# Minia React Project Context Contract

Use this skill when creating, validating, or repairing `AGENTS.md` for React-based Minia projects. It defines the React-specific context needed before `minia-fullstack` delegates milestone work.

## React Project Detection

Apply this skill when the project is any of:

- React app
- React Router app
- TanStack Router app
- TanStack Start app
- Dashboard
- Landing page
- Web frontend
- Full-stack app with React UI

If the project is not React-based, ignore this skill.

## Relationship To Base Project Context

This skill is additive. It extends only React-specific `AGENTS.md` requirements for `minia-fullstack`. Do not duplicate, replace, or weaken the base project context contract, base product-doc requirements, base command requirements, or base milestone/plan path requirements.

Apply this skill after the base project context checks identify a React-based project, then require the additional React fields below before delegation.

## Required AGENTS.md Fields

A React-based project context is complete only when `AGENTS.md` defines these fields with project-relative paths or explicit `none` values where allowed.

### Project Identity

- Project type identifies the React stack or web frontend type.
- Product name.
- Repository name.
- Primary domain terms.

### Product Docs

- PRD.
- TRD.
- UI/UX Design for frontend, dashboard, landing page, mobile, or full-stack web app work.

### Stack And Structure

- Programming language.
- Framework/project structure with the specific React stack.
- Package manager.
- Deployment target or build artifact.
- Database when the app has persistence.

### Canonical Paths

- Source root.
- Route/page modules.
- Components.
- Hooks or client state.
- Server functions, loaders, actions, API routes, or adapters when present.
- Schemas or validation modules when present.
- UI entrypoints.
- Interface entrypoints.
- Unit tests.
- E2E tests when required.
- Test helpers.
- Fixtures, factories, and mocks when present.
- Global styles or theme tokens.
- Router config.
- Generated route tree or generated router artifacts, or `none`.
- Build/config files relevant to React execution.

### Test Layout

- Unit tests path.
- E2E tests path when `E2E required: true`.
- E2E required: `true` or `false`.
- React Doctor command or package script.
- Accessibility, visual, integration, smoke, or public-interface layers when used.
- Shared test setup path when present.

### Commands

- Install.
- Format.
- Format check.
- Lint.
- Typecheck.
- Unit tests.
- E2E tests when required.
- Build.
- React Doctor.

React Doctor must run a full non-interactive scan, fail on warnings, and be treated as failed when it reports any errors, warnings, notes, diagnostics, or non-clean scores.

### React Parallel Planning Policy

- Max parallel subagents.
- Known shared files that should serialize work.
- Generated files policy: whether generated route trees or router artifacts may be edited directly, generated by commands only, or never edited.
- Snapshot and visual baseline policy when those files exist.
- E2E state isolation policy when E2E tests mutate shared app state.
- Whether route/page slices may be planned independently.
- Whether component slices may be planned independently.
- Whether data hooks, loaders, actions, server functions, and adapters may be planned independently.

## Context QA Behavior

If a required React field is missing, ask only for the missing field or infer it from explicit project files when safe. Do not invent conventions that are absent from the repository and not listed by the user.

Ask before adding placeholder docs, new test layers, new generated-file policies, or new command names.

Do not delegate to `minia-manager` until required React context is complete enough for planner and dispatch skills to produce exact target files and tests.

## Incomplete Context Blockers

Return or report a blocker instead of delegating when:

- React Doctor is missing.
- Unit test path or command is missing.
- `E2E required` is not explicitly true or false.
- E2E is required but no E2E path or command exists.
- Route/page and component paths are unknown.
- Generated file policy is unknown for router artifacts that exist.
- Max parallel subagents or shared-file policy is unknown and the milestone asks for parallel React execution.

Use `code: MISSING_CONTEXT` and name the missing field(s).
